# Phase: HUNT

## Purpose
Investigate, gather context, understand the problem before acting.

## Entry condition
- User request received but context insufficient for implementation
- Ambiguity in requirements
- Unknown codebase area

## Exit condition
- Clarity achieved: specific file paths, root cause, or design options documented

## Behaviors
- Read relevant source files
- Search codebase for patterns
- Check existing tests
- Read docs/research online if needed
- Ask user for clarification when stuck
- Update BOARD.md with findings if a new ticket emerges

## Output
- Ticket added to BOARD.md (if needed)
- Findings documented in LOG.md
- Clear path forward
